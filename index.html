Closures in functional programming

Closures occur when a nested function has access to the variables of its containing (enclosing) function. They are often used for maintaining state and encapsulation.

Closures are a way to create and use functions that "remember" the data in their containing scope.

Lunar Descent Navigation

How Closures Work in This Context:

  • Creation of the Closure: The function create_navigation_algorithm creates a closure named navigate. This closure contains the algorithm for adjusting the descent rate based on sensor data.
  • Access to Outer Scope Variables: The navigate function has access to the sensor_data variable defined in the outer scope of create_navigation_algorithm. This is achieved using the nonlocal keyword.
  • Encapsulation: The navigate function encapsulates the navigation logic. It can access and modify the sensor_data variable, but the details of how the navigation algorithm works are hidden within the closure.
  • Retaining State: Each time navigate is called, it updates the sensor_data variable and calculates the descent rate based on the updated data. The closure "remembers" the previous state of sensor_data across multiple calls.
  • Autonomous Navigation: During the lunar descent loop, the closure is invoked repeatedly to get the descent rate. The algorithm adapts to changes in altitude and velocity, allowing the spacecraft to autonomously adjust its trajectory in real-time.
  • Continuation of State: Even though the navigate function is called multiple times within the loop, it retains access to the same sensor_data variable. This allows it to continue updating the descent rate based on the latest sensor readings.
